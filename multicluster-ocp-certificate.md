# Multicluster issue with certificate 

When you set up multicluster you need communication to be secure on both side :

- From primary to secondary for click through, the primary need to trust the ca certificate of each of his secondaries 
- From secondary to primary for licenses leasing each secondary need to trust the ca certificate of the primary

## Discover how openshift router CA certificate are managed on your cluster 

https://docs.redhat.com/en/documentation/openshift_container_platform/4.21/html/security_and_compliance/configuring-certificates#understanding-default-ingress_replacing-default-ingress

if 
```
oc get proxy/cluster -o jsonpath='{.spec.trustedCA.name}'
```

Return nothing you are using the default router CA certificate generated by the ingress controller. 
Otherwise you use a custom CA certificate.


### If you work with the default router CA certificate generated at install 

On the primary 
```
oc get secret router-ca -n openshift-ingress-operator -o jsonpath='{.data.tls\.crt}' | base64 --decode > primary-ca-bundle.pem
``` 

Repeat this operation on each secondary (notice that we concat the certificate with `>>`)
```
oc get secret router-ca -n openshift-ingress-operator -o jsonpath='{.data.tls\.crt}' | base64 --decode >> secondaries-ca-bundle.pem
``` 

### if you Work with a custom CA authority 


Find out the internal CA 

```
cert=$(oc get proxy/cluster -o jsonpath='{.spec.trustedCA.name}')
```

On the primary 
```
oc get cm $cert -n openshift-config -o jsonpath='{.data.ca-bundle\.crt}' > primary-ca-bundle.pem
```

Repeat this operation on each secondary (notice that we concat the certificate with `>>`)
```
oc get cm $cert -n openshift-config -o jsonpath='{.data.ca-bundle\.crt}' >> secondaries-ca-bundle.pem
```

## Now create the config map in the clusters

https://docs.kasten.io/latest/install/advanced#install-root-ca-in-veeam-kastens-namespace

### On the primary 

```
oc --namespace kasten-io create configmap custom-ca-bundle-store --from-file=secondaries-ca-bundle.pem
```

The primary will add all the ca certificate of the secondaries

### On each secondaries 

```
oc --namespace kasten-io create configmap custom-ca-bundle-store --from-file=primary-ca-bundle.pem
```

## Adapt Kasten configuration 


on the primary 
```
cacertconfigmap:
  name: custom-ca-bundle-store
  key: secondaries-ca-bundle.pem
```

on each secondaries
```
cacertconfigmap:
  name: custom-ca-bundle-store
  key: primary-ca-bundle.pem
```
